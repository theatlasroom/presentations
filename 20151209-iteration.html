UI Design

  • I'm not a visual artist, so I prefer to start digitally instead of paper
  • If i'm totally lost on what I want to acheive I'll start from paper, so I dont waste time pushing pixels
  • Today i'll focus on responsive UIs - I won't cover mobile specific UIs
  • A little bit of thought put into your ui up front will save you a lot of time in implementation

UI - Moqups

  • moqups.com
  • Wireframing + interaction design
  • Focused on nailing down the layout and basic flow of the interface - forget about the particulars
  • Drag and drop interface with all your common web UI components - collaboration friendly
  • User testing / validation
